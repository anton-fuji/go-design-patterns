# Strategy Pattern
- アルゴリズムをオブジェクトとしてカプセル化し、実行時に切り替え可能にする振舞関連のデザインパターンの1つ

| 特徴 | Before (条件分岐) | After (Strategyパターン) |
| :--- | :--- | :--- |
| **拡張性** | 新機能追加のたびに `Cache` を修正 | ファイルを追加するだけ |
| **コードの見通し** | 巨大な `switch` 文で読みづらい | 各ファイルが小さくシンプル |
| **テスト** | 特定のロジックだけテストするのが難しい | 戦略ごとに単体テストが可能 |
| **結合度** | `Cache` が全アルゴリズムに依存（密結合） | `Cache` はインターフェースのみ依存（疎結合） |

# ファクトリメソッドとは？
オブジェクトの生成処理をカプセル化し、クライアントコード（呼び出し元）が具体的なクラス名を知ることなく、
抽象的なインターフェースを通じてオブジェクトを取得できるようにする構造的なデザインパターンです。



# 今回のサンプルについて
## 多種の通知サービスについて考えてみる
ファクトリ関数 GetNotifier を使って、Eメール通知・SMS通知 など複数の通知サービスを生成していきます。

ファクトリパターンを用いることで、「疎結合の実現」・「オブジェクト生成の集中管理」が可能になります。
**疎結合の実現**
- クライアントコード（例：main関数）は、EmailNotifierやSMSNotifierといった具体的なクラスの実装に依存しません

**オブジェクト生成の集中管理**
- 新しい通知手段を追加する場合でも、クライアントコードを修正する必要はなく、ファクトリ関数（GetNotifier） の内部のみを変更すれば済みます
